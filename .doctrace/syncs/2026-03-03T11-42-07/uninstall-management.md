## Changes Applied

### 1. Fixed hook marker in standalone hook example (line 72)
- **Was:** `# branch-ctx`
- **Now:** `# branch-ctx-managed`
- **Reason:** `HOOK_MARKER` in `src/branchctx/constants.py` is defined as `"# branch-ctx-managed"`. The `_is_standalone_bctx_hook()` function in `hooks.py` checks `lines[1].strip() == HOOK_MARKER` to identify standalone hooks. The doc showed the wrong marker string, which would mislead anyone trying to understand hook detection.

### 2. Fixed hook marker in appended hook example (line 87)
- **Was:** `# branch-ctx`
- **Now:** `# branch-ctx-managed`
- **Reason:** Same as above. The append snippet generated by `_get_append_snippet()` uses `HOOK_MARKER` (i.e., `# branch-ctx-managed`) as the start delimiter and `SNIPPET_END_MARKER` (`# branch-ctx-end`) as the end delimiter. The doc showed the wrong start marker.
